# git init
  -it turns the folder into git repository.which means it is used to initialize a new Git repository
  -once the git init command is typed it initializes the folder as a Git repository by creating a .git directory, where git store all version control data and history
  -the directory is hidden , to check that type ls -a command in the cmd

# git status

==============================================================================================================================================================

# git add 
  -the git add command is used to move selected change from the working directory to the staging area(index) so they can be include in the next commit.
  -so now we can get doubt that what does the staging area means, so we will try to understand it easily that staging area is also called as index , it is an intermediate layer in git where you prepare and review changes before you commit them to the repository
  -to select all file use 
    # git add . 
  -to select a particular file use 
    # git add filename.txt filename.html filename.js
  -after we add the file to the staging area we got the below output
    # On branch master
      No commits yet
      Changes to be committed:
        (use "git rm --cached <file>..." to unstage)
              new file:   README.md
              new file:   index.html
              new file:   script.js
              new file:   style.css
  -here we can see the changes that the file is moved to the staging area and is ready to next commit
  -and at last we got a hint message which is (use "git rm --cached <file>..." to unstage) here it states that if you added any file by mistake you can remove that file from the staging area without deleting it
  
==============================================================================================================================================================  

# once we moved the file to the staging are we got two options either we can commit the file which means we can move the file to tracked area OR we can bring it back to untrack area which means unstaging
   
   git rm --cached filename.txt
   -this command is used to unstage a single file
   
   git rm --cached filename -r .
   -this command is  used to unstage all the file in the directory
   
   here - git rm = by default this command deletes files from your both your git repository and actual computer drive 
        - --cached = this flag is saftey net. this tell git to only remove the files from git's tracking index(staging area). it ensures your actual file on your computer are not deleted.
		- -r = this stands for "recursive". it allows the git to look inside the currect folder and apply the command to every single file and subfolder inside it.
		- . = this represents the current directory(the folder your are currently in your directory)
  
===============================================================================================================================================================

# now instead of untracking if we want to move the file to tracked area. so we will below command
  git commit -m "message"
  -saves a permanent snapshot of the staged changes into the Git repository with a message
  -in simple words 'git add select the file' and 'git commit saves them'
  -the -m means message it is not compulsory to write message but it is mandatory to write a message which can be helpful later
  -once the git commit command is entered it  takes files from the staging area and creates a snapshot and stores inside the .git/object directory
  -Important points
    Commits are permanent records
    Git commits only what is staged
    Unstaged files are ignored
    Each commit represents a stable state

# flow chart till now what we have learned

[ Working Directory / untracked] -> (Modify files)
        │
        ▼  git add <filename>
[ Staging Area ]      -> (Prepare snapshot)
        │
        ▼  git commit -m "your message"
[ Local Repository / tracked]  -> (Permanently save to history)

# git remote add origin https://github.com/USERNAME/REPO_NAME.git

  -git
     Calls the Git program
   remote
     Tells Git you’re working with a remote repository
     A remote repo = repository hosted somewhere else (GitHub, GitLab, etc.)
   add
     Means add a new remote connection
   origin
     This is just a name (alias) for the remote repository
     Conventionally, the main remote is called origin
     You could name it anything, but origin is standard

# git remote -v
  =it is used to view the remote repository linked to your local Git repository along with their URLs
  -in simple words It tells where your code will be pushed to and pulled from 
  -here -v = verbose which means it show both fetch and pull URLs
  -when you run the git remote -v command you will get output like mentioned below 
  -origin  https://github.com/USERNAME/REPO.git (fetch)
   origin  https://github.com/USERNAME/REPO.git (push)
  -here
    origin → remote name
    fetch → used when you pull or fetch code
    push → used when you push code
